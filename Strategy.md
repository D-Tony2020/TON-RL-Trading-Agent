## MDP 设计决策

### 状态空间

| Agent | 维度 | 特征 | 状态数 |
|-------|------|------|--------|
| Q-Learning | 6维离散 | price_change_24h, price_change_4h, volatility_24h, rsi_14, hour_of_day, position | 4,480 |
| DQN | 7维连续 | 上述6维 + volume_ratio, 归一化到 [0,1] | 连续 |

**设计理由**:
- Q-Learning 用离散状态是表格法的固有要求, 4,480 states 对 17K 数据量合理
- DQN 额外加入 volume_ratio 因为神经网络能处理连续值, 且成交量是重要的市场微观结构信号
- 未将跨资产特征 (BTC/ETH) 纳入状态: 避免时间对齐复杂度, 且 7 维对数据量已足够, 跨资产分析作为独立模块呈现

### 动作空间

**Run #1-#2 (3动作)**:
```
A = { BUY(0), HOLD(1), SELL(2) }
position ∈ {0, 1}, 不做空
```

**Run #3+ (5动作, 当前版本)**:
```
A = { BUY(0), HOLD(1), SELL(2), SHORT(3), COVER(4) }
position ∈ {-1, 0, 1}
```
- 全仓操作: BUY=空仓→多仓, SELL=多仓→空仓, SHORT=空仓→空头, COVER=空头→空仓
- 非法动作映射 → HOLD (例: 多仓 SHORT, 空头 BUY, 空仓 SELL/COVER)

### 奖励函数

两种模式, 可配置:

| 模式 | 公式 | 用途 |
|------|------|------|
| `simple` | position * hourly_return - cost_rate * \|position_change\| | Q-Learning 默认, 公平对比基线 |
| `sharpe` | simple + lambda_sharpe * differential_sharpe - lambda_risk * drawdown_penalty | DQN ablation study |

当前全部训练使用 `simple` 模式, 保证 Q-Learning 与 DQN 的公平对比。

### Episode 设计

- 每个 episode = 随机采样连续 720 小时 (约 30 天)
- 起始点从训练集中随机选取
- 优势: 多次曝光不同 regime, 长度可控, 加速收敛

## Run #2: 修复 epsilon 衰减率

**日期**: 2026-03-05
**改动**: 校准 epsilon_decay 使得训练结束时 epsilon 恰好达到 epsilon_min (0.05)

### 工程改动

```
Q-Learning: epsilon_decay  0.9995 -> 0.9925  (0.9925^400 = 0.049)
DQN:        epsilon_decay  0.9995 -> 0.985   (0.985^200  = 0.049)
```

### 工程参数

| 参数 | Q-Learning | DQN |
|------|------------|-----|
| alpha / lr | 0.1 | 0.0003 (Adam) |
| gamma | 0.97 | 0.99 |
| epsilon_decay | **0.9925** / episode | **0.985** / episode |
| n_episodes | 400 | 200 |
| episode_length | 720 | 720 |
| 训练结束 epsilon | **0.0500** | **0.0500** |
| 训练时间 | 75.0s | 702.1s |
| Q-table / Buffer | 1,708 states | 100,000 transitions |

### 交易策略分析

**DQN 的成功策略 (-25.28% vs Buy & Hold -52.05%)**:
1. **选择性入场**: 214 笔交易 (vs Buy & Hold 的 1 笔, Random 的 1123 笔), 在 3,435 步测试中仅 6.2% 的时间做出交易决策
2. **风险控制**: 最大回撤仅 35.98%, 远低于 Buy & Hold 的 57.35%, 说明 Agent 学会了在下跌前离场
3. **低 Win Rate 但高质量**: 18.05% 的胜率配合 -25.28% 的总亏损 (vs 市场 -52%), 说明虽然多数单笔交易亏损, 但盈利交易的收益远大于亏损交易的损失
4. **HOLD 主导策略**: 在下跌市场中, 空仓 HOLD (不入场) 本身就是正确策略; Agent 学会了 "不做就是赚"

**Q-Learning 的局限**:
1. 虽然也优于 Buy & Hold, 但 Sharpe (-2.92) 差于 Buy & Hold (-1.84), 说明交易频率带来了额外波动
2. 276 笔交易 vs DQN 的 214 笔, 交易成本更高
3. 表格方法的泛化能力有限: 测试集中的状态组合可能在训练中未充分覆盖

## Run #3: 引入做空动作 (SHORT/COVER)

**日期**: 2026-03-05
**改动**: 动作空间从 3 扩展到 5, 引入 SHORT(做空) 和 COVER(平空), position ∈ {-1, 0, 1}
**Git**: `main` 分支, commit `b39e09a`

### 设计变更

| 变更项 | Run #2 | Run #3 |
|--------|--------|--------|
| 动作数量 | 3 (BUY/HOLD/SELL) | **5 (BUY/HOLD/SELL/SHORT/COVER)** |

### 交易策略深度分析

**Q-Learning 的成功 (从 -47% 到 +61%)**:
1. **做空获利**: 在 decline regime 中 SHORT 占比 5.5% (远高于其他 regime 的 2.4-2.6%), 说明 Agent 学到了在下跌趋势中做空获利
2. **交易对平衡**: SHORT (5.5%) ≈ COVER (5.3%), 做空后能及时平仓, 非单方面押注
3. **Regime 差异化**: decline 时 HOLD 仅 82.7% (vs rise/steady 的 92%), Agent 在下跌期间更积极交易
4. **353 笔交易 / 3,434 步 = 10.3% 交易率**: 适度活跃, 既能捕捉机会又不被交易成本拖累
5. **Win Rate 46.71%**: 接近 50%, 说明策略并非靠少数大赢, 而是持续的小幅获利
6. **Sharpe +1.93**: 这是所有策略中唯一正 Sharpe, 表明风险调整后的正回报

**为什么 Q-Learning 表现超越 DQN?**:
1. **状态覆盖率**: Q-table 2,547 / 6,720 = 37.9%, 大部分常见市场状态已有足够经验
2. **表格法的确定性**: Q-Learning 对已访问状态给出确定性 Q 值, DQN 的网络近似可能在分布外状态上泛化不佳
3. **5 动作下的探索**: DQN 仅 94 笔交易 (2.7% 交易率), 过于保守; Q-Learning 的 369 笔 (10.7%) 更好地利用了做空机会
4. **DQN 的 "做空恐惧"**: DQN 在 decline 中 SHORT 仅 2.5%, Q-Learning 达 5.9%; DQN 可能因网络泛化导致对 SHORT 的 Q 值估计偏保守

**RSI Rule Baseline 的有趣改善** (-44.38% → -35.20%):
- 引入做空后 RSI Rule 多了 SHORT (RSI>70 且空仓) 和 COVER (RSI<30 且空头) 两个动作
- 从 -44.38% 改善到 -35.20%, 做空本身就为规则策略带来了 +9pp 的提升
- 37 笔交易中包含做空操作, 说明 RSI 规则在极端超买时做空是有效的

### 历次教训总结

1. **Epsilon 衰减率必须匹配训练预算** (Run #1→#2): per-episode 衰减率需要精确校准, 确保训练结束时 epsilon 达到 epsilon_min。公式: `decay = epsilon_min^(1/n_episodes)`
2. **数据对齐要处理边界情况**: SPY 半小时偏移 (14:30, 15:30) 导致 inner join 零行, 需 resample 到整点
3. **下跌市场中 "不做" 就是好策略** (Run #2): Agent HOLD 占比高 (92-94%) 并非失败, 而是在下跌趋势中的合理行为
4. **做空是下跌市场的 game changer** (Run #2→#3): 3 动作下最佳策略是 "不入场" (上限 0%); 5 动作下可以从下跌中获利 (Q-Learning +61.07%), 理论上限大幅提高
6. **Greedy action 的确定性至关重要** (Run #3 修复): Q-Learning 随机平局打破导致回测结果在 +24%~+81% 间波动, 改用 argmax 后固定为 +61.07%
